## What shipped

**Bug:** Lines 384–387 of `src/app/dashboard/campaigns/campaigns-client.tsx` wired every row `<tr onClick>` to open the rename-only edit modal. The 5-tab detail page (Analytics / Leads / Sequences / Schedule / Options) shipped by CC #UI-3 (PR #51) and CC #UI-3-rev (PR #53) was reachable only by typing the URL — there was zero in-UI path to it.

**Fix (one file, +18/−3 LOC):**
1. Added `Pencil` to the existing `lucide-react` import.
2. Added `const router = useRouter();` (the import for `useRouter` was already present but unused).
3. Added a new "Actions" `<th>` column header.
4. Replaced the row `onClick` body with `router.push(\`/dashboard/campaigns/${campaign.id}\`)`.
5. Added a trailing `<td>` per row containing a Pencil-icon `<button>`. Its `onClick` calls `e.stopPropagation()` then `setEditingCampaign(campaign)` + `setModalOpen(true)`. Has `aria-label="Edit campaign"`.

**No other source files touched.** No migrations. No API changes. No sender-pipeline edits.
