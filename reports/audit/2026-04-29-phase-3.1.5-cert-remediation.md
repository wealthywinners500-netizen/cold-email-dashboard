All 6 secrets handled in Python process memory only — never logged to chat, never written to disk, never serialized into reports.

3. AES-256-GCM decrypt each ciphertext using the pattern in `src/lib/provisioning/encryption.ts` (IV[12] || ciphertext || authTag[16], base64-encoded). All 4 root passwords decoded, length=22.
4. Supabase REST GET `email_accounts?server_pair_id=eq.<P11_ID|P12_ID>&status=eq.active&order=email.asc&limit=1&select=email,smtp_pass,smtp_user` → P11 test account `alexander.black@caliri.info`, P12 test account `angela.bailey@lausha.info`.
