## P11 cert detail (additional evidence)

Probed P11 server (69.164.210.213) with three different SNI hints — all returned the same single cert:

```
SNI=mail.caliri.info       → CN=mail1.caleong.info  (SAN: mail1.caleong.info ONLY)
SNI=mail1.caleong.info     → CN=mail1.caleong.info  (same cert)
SNI=mail.voilit.info       → CN=mail1.caleong.info  (same cert)
```

The server has no SNI-aware cert routing — single cert with single DNS SAN. Per Dean's per-domain SMTP/IMAP architecture directive (2026-04-28), the expected state is a per-domain cert per sending_domain. This is real engineering work to remediate: issue 10 per-domain LE certs + configure Postfix/Exim SNI-aware cert serving.
