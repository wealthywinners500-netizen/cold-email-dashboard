## Generation methodology (paste-ready reproducible recipe)

For each pair the generator:

1. Queries Supabase REST `email_accounts?server_pair_id=eq.<id>&status=eq.active&select=email,smtp_user,smtp_pass&order=email.asc`. Deterministic ORDER BY `email asc` ensures stable row order.
2. For each row:
   - `From Email*` ← `email`
   - `From Name*` ← local-part of email split on `.`, each segment `str.capitalize()`, joined with space (e.g., `alex.thompson` → `Alex Thompson`).
   - `Username` ← `smtp_user` (Dean enhancement (2); equals email in practice for every row).
   - `SMTP Host*` ← `mail.<sending_domain>` where sending_domain is the part after `@` in email.
   - `SMTP Port*` ← `587`, `SMTP encryption` ← `TLS`.
   - `SMTP Password*` ← `smtp_pass` (plaintext; saga writes already-decrypted per F-9).
   - `Reply-to email address` ← empty.
   - `IMAP Host*` ← `mail.<sending_domain>` (same per-domain host).
   - `IMAP Port*` ← `993`, `IMAP encryption` ← `SSL`.
   - `IMAP Username` ← `smtp_user`.
   - `IMAP Password` ← `smtp_pass`.
   - `Delay between emails` ← `random.Random("20260429-pair<N>").randint(420, 720)` per row (seeded once per pair; subsequent draws produce the table's deterministic distribution).
   - `Daily Limit` ← `30`.
3. Writes `snovio_csvs_v3/pair_<N>_<ns_domain>_2026-04-29.csv`:
   - Line 1: `# Total: <N> accounts\n` (LF, matches canonical)
   - Line 2: 207-byte header `From Email*,…,Daily Limit\r\n` (CRLF, byte-for-byte canonical)
   - Lines 3+: 15-column data rows with CRLF terminators
   - RFC 4180 quote-escape: a field containing `,`, `"`, `\r`, or `\n` is wrapped in `"…"` with interior `"` doubled. (No row in this run triggered escape — all values are alphanumeric + `.` + `@` + `-`.)
4. Writes adjacent `pair_<N>_<ns_domain>_2026-04-29.metadata.md` with: generated-at, source row count, RNG seed, output CSV path, SHA-256, sending-domain coverage, delay distribution, pre-import gate evidence reference, Snov.io reset-risk readout (always N/A this run — no prior P11–P21 Snov tags exist), header byte-for-byte spec, reproducibility recipe.
