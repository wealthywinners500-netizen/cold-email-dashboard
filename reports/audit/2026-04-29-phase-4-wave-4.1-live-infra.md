### Drift categories

1. **F-1 P1 (carries from Phase 0):** `DATABASE_URL` missing from Vercel prod. Single-line fix at Phase 8.
2. **P2 naming inconsistency:** IONOS (`API_KEY` vs `API_PREFIX`+`API_SECRET`) and LINODE (`API_TOKEN` vs `API_KEY`) use different env-key NAMES on Vercel vs worker. Code likely paths through `process.env.IONOS_API_KEY` on Vercel and `process.env.IONOS_API_PREFIX` + `_SECRET` on worker. If a future code path tries to read both from a shared module, it breaks. Wave 4.2.a (codebase grep) will surface which names appear where.
3. **P2 architectural verify:** `NAMECHEAP_*` on Vercel but absent from worker ‚Äî is the provisioning saga's domain-registration step running on the worker (without these keys) or being delegated back to Vercel? Phase 5 / Phase 7 trace.

## Wave 4.2 forward-looking decision

### Clerk auth path for 4.2.c UI walk

Per Dean's nudge, dashboard pages with user-scope binding (`/dashboard/servers/[id]`, `/dashboard/campaigns`, etc.) will 401 on plain HTTP with a dev cookie since Clerk's `auth.protect()` is server-side and reads from the session cookie + JWT.

**Decision:** Wave 4.2.c will use **Chrome MCP** (`mcp__Claude_in_Chrome__*` ‚Äî load via ToolSearch at the start of 4.2.c) against Dean's existing browser session.

Reasoning:
- Dean's local browser already holds a valid Clerk dev-instance session (charming-dane-14) via cookies + localStorage.
- Chrome MCP attaches to the running browser, inherits the session ‚Äî no JWT extraction or `Authorization` header binding needed.
- For UI bugs (broken renders, console errors, count mismatches), the DOM-aware tooling is faster + more reliable than headless HTTP+screenshot.
- Read-only by default; no clicks that mutate (we only `mcp__Claude_in_Chrome__navigate` + `read_page` + `read_console_messages` + `find` ‚Äî no form-submits or button-clicks unless explicitly authorized for a specific check).

The user-explicit fallback (dev JWT bound via Authorization header) stays available if the browser MCP is unreachable ‚Äî but is more brittle for client-side rendered pages.
