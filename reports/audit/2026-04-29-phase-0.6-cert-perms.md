### Security Issues (on P13 — only auditable pair)

**Finding #2: Domain Cert Keys World-Readable (20 files)**
- **Severity:** P1 (not P0, as HELO keys are correct)
- **Affected Files:** All per-domain cert keys on P13 (both servers)
  - P13 S1: `/home/admin/conf/mail/{caleap,carena,cereno,cerone,corina}.info/ssl/*.key` (10 files)
  - P13 S2: `/home/admin/conf/mail/{larena,seamle,seapot,searely,voility}.info/ssl/*.key` (10 files)
- **Current Mode:** 0644 (owner read/write, group read, **world read**)
- **Expected Mode:** 0640 (owner read/write, group read, world: none)
- **Impact:** Unauthorized users on the system can read private TLS keys, risking domain spoofing/interception
- **Remediation:** 
  ```bash
  chmod 640 /home/admin/conf/mail/*/ssl/*.key
  chown admin:mail /home/admin/conf/mail/*/ssl/*.key
  ```

Surfaced as P3 backlog only — the saga's `chmod 0640` enforcement (HL #145, gate-tested) covers HOSTNAME certs going forward; per-domain template needs a separate one-shot sweep.

## Audit Methodology

**Command Run:**
```bash
echo "=== HELO host cert ==="
stat -c "%a %U:%G %n" /usr/local/hestia/ssl/certificate.crt /usr/local/hestia/ssl/certificate.key 2>&1
echo "=== Per-domain mail certs ==="
for D in $(ls /home/admin/conf/mail/ 2>/dev/null); do
  for F in /home/admin/conf/mail/$D/ssl/$D.crt /home/admin/conf/mail/$D/ssl/$D.key; do
    [[ -f "$F" ]] && stat -c "%a %U:%G %n" "$F"
  done
done
```

**SSH Connection:**
```bash
ssh -o ConnectTimeout=8 -o BatchMode=yes -o StrictHostKeyChecking=accept-new root@<ip> <cmd>
```
