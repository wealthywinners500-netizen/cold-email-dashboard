Direct ground-verify of `src/app/dashboard/campaigns/campaigns-client.tsx` on `origin/main`:

- Lines 382–388: every campaign-row `<tr>` wires `onClick` to `setEditingCampaign(campaign)` + `setModalOpen(true)`. This opens the existing edit-name/region/store-chain modal.
- `useRouter` IS imported on line 4 — but the hook is **never called** and `router.push` does not appear anywhere in the file (`grep -n "router" → no matches`).

## 2. Fix design

Single file: `src/app/dashboard/campaigns/campaigns-client.tsx`

1. Add `Pencil` to the existing `lucide-react` import (line 7). `useRouter` already imported.
2. Add `const router = useRouter();` next to the existing `useState`/`useRealtimeRefresh` hooks at top of `CampaignsClient`.
3. Add an "Actions" column header (a new `<th>` after Status, line ~377).
4. Change the row's `onClick` (currently lines 384–387) to `router.push(\`/dashboard/campaigns/${campaign.id}\`)`.
5. Add a new trailing `<td>` per row containing a `<button>` with the `Pencil` icon. Its `onClick` calls `e.stopPropagation()` then `setEditingCampaign(campaign)` + `setModalOpen(true)`. `aria-label="Edit campaign"`, `title="Edit campaign"`.
6. No other rows have action buttons (no existing delete button on the row), so no other `e.stopPropagation()` additions needed in row body cells.

The two table-rendered `<button>` toggles already outside the row (Create Campaign on line 67 + line 179) keep their existing `setEditingCampaign(null) + setModalOpen(true)` flow — they create new campaigns, not edit existing ones, so they don't bubble through the new nav.
