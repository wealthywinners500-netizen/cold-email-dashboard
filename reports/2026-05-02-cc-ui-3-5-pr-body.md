## Summary

**Bug:** the campaigns LIST page (`/dashboard/campaigns`) wired every row click to open the rename-only edit modal. The 5-tab Instantly-style detail page shipped by CC #UI-3 (#51) and CC #UI-3-rev (#53) was reachable only by typing the URL — there was zero UI path from the list to it.

**Fix (one file, +18/−3 LOC):** `src/app/dashboard/campaigns/campaigns-client.tsx`
- Row `onClick` now calls `router.push(\`/dashboard/campaigns/${campaign.id}\`)`.
- Edit modal preserved via a new explicit Pencil-icon button in a new trailing "Actions" `<td>`. Its `onClick` calls `e.stopPropagation()` then `setEditingCampaign(campaign)` + `setModalOpen(true)`.
- Used the already-imported (but previously unused) `useRouter` from `next/navigation` — added the hook call.
- Added `Pencil` to the existing `lucide-react` import.

## Why

After this PR, clicking any campaign row in the list navigates to the 5-tab detail page (Analytics / Leads / Sequences / Schedule / Options). The rename-only edit modal is still accessible — it now lives behind an explicit pencil icon so it doesn't compete with row-level navigation.
