## Appendix A — Observed anomalies + deferred items

1. **`server_pairs.status` = `active`, not `live`.** Prompt repeatedly refers to `status='live'`; the enum's active-state value is `active`. Previously flagged in `2026-04-22-p14-retry-halt.md`. No saga change needed; this report uses the schema's value.
2. **DKIM canonical selector is `mail`, not `s1`/`s2`.** Prompt's Part 6 cites HL #51 but uses `s1._domainkey` / `s2._domainkey` as the dig targets. The shared-key-scp pattern makes `mail._domainkey.<domain>` resolve identically on both servers — this is the correct target. Prompt drift to flag for Dean at next prompt-template review.
3. **Step 6 (`set_ptr`) duration dropped 642s → 21s** between the 04-21 cancelled job and this one. Confirmed design change (fire-and-forget Linode API, propagation deferred to VG2's fcrdns check). The PR #16 retry-with-backoff is the catch-net for that deferral; it wasn't needed this run because PTR propagated within the ~24-min window between `set_ptr` and VG2.
4. **Auto-fix attempted 13, "12 fixed, 0 failed".** One item self-resolved during the post-fix zone sync. Worth opening a pointer in the saga logs to distinguish `self_resolved_by_sync` from `fixed_directly` on a future cleanup pass — not a blocker.
5. **P13 row missing from `project_server_deployment.md`.** Chronological state-file entries stop at P12 despite P13 (launta.info) being live since 2026-04-19. Memory structure gap — this report adds P14, and the consolidate-memory pass should backfill P13 from its 04-19/04-20 reports.
6. **Mail-tester + Postmaster signals deferred.** See Part 5 (b)/(c). Dean to probe during Day-1 warm-up. Day-0 gate does not require these; verify-zone FAIL=0 + dig factual ✓ + canonical DMARC end-to-end is the accepted canonical triple per HL #110.
